---
title: "Vault — Storage Service"
description: Path-based file storage on the local filesystem with per-file public/private access control.
---

Vault stores arbitrary files for your apps — documents, images, reports — scoped per app and accessible through the standard API gateway.

## Core concepts

### Collections

Files are organised into **collections** — developer-defined namespaces within an app (think folders). A collection is just a string in the path; no setup required.

```
/patient-records/john-doe-xray.jpg
/invoices/2026-01.pdf
/avatars/user-42.png
 ↑         ↑
 collection filename
```

### Public vs private

Every file has an `is_public` flag (default: `false`). Private files require an API key. Public files are accessible by anyone at a stable URL with no credentials.

---

## API

All authenticated requests go through the Gateway at `api.{org}.local`. Include your API key:

```
Authorization: Bearer nbl_<key_id>.<secret>
```

### Upload a file

```
POST /v1/vault/files/{collection}/{filename}
Content-Type: multipart/form-data

file=<binary>
```

- Returns `409 Conflict` if a file already exists at that path. Delete it first or use a different name.
- Returns `413` if the file exceeds the app's size limit (default 50 MB).
- Returns `415` if the file extension is not in the app's allowed list.

**Response:**
```json
{
  "ok": true,
  "fileId": "d4e5f6a7-...",
  "path": "/patient-records/john-doe-xray.jpg",
  "isPublic": false,
  "publicUrl": null
}
```

### List files

```
GET /v1/vault/files                    # all files for the app
GET /v1/vault/files/{collection}       # files in a specific collection
```

**Response:**
```json
{
  "ok": true,
  "files": [
    {
      "id": "d4e5f6a7-...",
      "collection": "patient-records",
      "filename": "john-doe-xray.jpg",
      "mimeType": "image/jpeg",
      "sizeBytes": 204800,
      "isPublic": false,
      "publicUrl": null,
      "createdAt": "2026-05-30T10:00:00Z"
    }
  ]
}
```

### Download a file

```
GET /v1/vault/files/{collection}/{filename}
```

Returns the raw file bytes with the correct `Content-Type` header. Private files require a valid API key; public files can be fetched from the public URL instead.

### Toggle public access

```
PATCH /v1/vault/files/{collection}/{filename}
Content-Type: application/json

{ "isPublic": true }
```

Once a file is public its URL is stable and permanent until you make it private again.

### Delete a file

```
DELETE /v1/vault/files/{collection}/{filename}
```

Permanently removes the file from disk and its metadata from the database. This cannot be undone.

---

## Public URLs

When `isPublic` is `true`, the file is accessible at:

```
http://api.{org}.local/vault/{app_slug}/{collection}/{filename}
```

No API key required. The Gateway forwards these requests to Vault, which checks the `is_public` flag before serving. If the flag is `false`, the response is `403 Forbidden` regardless of the URL.

## Client SDK — `@nublestation/vault`

`@nublestation/vault` is the file storage client for NubleStation apps. Works in the browser and Node.js.

### Install

```bash
npm install @nublestation/vault
# or
pnpm add @nublestation/vault
```

### Create a client

```typescript
import { createVaultClient } from "@nublestation/vault";

const vault = createVaultClient({
  url:    "http://api.clinic.local",  // Gateway base URL
  apiKey: "nbl_<key_id>.<secret>",    // API key from the Console
});
```

All calls are automatically scoped to the app that issued the API key.

### Key operations

```typescript
// Upload — from a browser file input or a Buffer/Uint8Array
const result = await vault.upload("patient-records", "xray.jpg", file);

// List a collection (or omit for all files)
const files = await vault.list("patient-records");

// Download as ArrayBuffer
const bytes = await vault.download("patient-records", "xray.jpg");

// Make public (returns stable URL) / private
const updated = await vault.setPublic("patient-records", "xray.jpg", true);
// Public URL: http://api.clinic.local/vault/{appSlug}/patient-records/xray.jpg

// Delete permanently
await vault.delete("patient-records", "xray.jpg");
```

### Ownership & sharing (ADR 016)

When combined with `@nublestation/identity`, every file is owned by the uploading user and can be shared per-user with `viewer` or `editor` roles:

```typescript
// Scoped listings
const mine   = await vault.listMine();
const shared = await vault.listSharedWithMe();

// Share with a specific user
await vault.share("reports", "q1.pdf", granteeUserId, "viewer");
await vault.unshare("reports", "q1.pdf", granteeUserId);
```

Access is enforced server-side — a `viewer` calling `delete()` receives `403`.

### Error handling

```typescript
import { VaultError } from "@nublestation/vault";

try {
  await vault.upload("docs", "report.pdf", bytes);
} catch (err) {
  if (err instanceof VaultError) {
    // err.status (HTTP code), err.code (machine string)
    if (err.status === 409) console.warn("File already exists");
  }
}
```

## App settings

In the Console under **App → Vault Settings**, admins can configure:

| Setting | Default | Description |
|---|---|---|
| Allowed extensions | _(all)_ | Whitelist of file extensions (e.g. `pdf, jpg, png`). Empty = allow all. |
| Max file size | 50 MB | Per-file upload limit. |

---

## Storage layout on disk

Files are stored on the NubleStation host at:

```
/var/nuble/storage/
  {app_slug}/
    {collection}/
      {filename}
```

Paths are validated server-side — path traversal attempts are rejected before any filesystem operation.

---

## Limits

- **No versioning.** A file at a given path is a single version. To update a file, delete it and re-upload.
- **No total quota** per app. Disk management is the host administrator's responsibility.
- **50 MB per file** by default, configurable per app in Console.
- **LAN only.** Files are not accessible from outside the organisation's network.
