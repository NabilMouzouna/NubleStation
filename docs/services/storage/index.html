Vault stores arbitrary files for your apps — documents, images, reports — scoped per app and accessible through the standard API gateway.

Core concepts

Collections

Files are organised into collections — developer-defined namespaces within an app (think folders). A collection is just a string in the path; no setup required.

/patient-records/john-doe-xray.jpg
/invoices/2026-01.pdf
/avatars/user-42.png
 ↑         ↑
 collection filename

Public vs private

Every file has an is_public flag (default: false). Private files require an API key. Public files are accessible by anyone at a stable URL with no credentials.

API

All authenticated requests go through the Gateway at api.{org}.local. Include your API key:

Authorization: Bearer nbl_<key_id>.<secret>

Upload a file

POST /v1/vault/files/{collection}/{filename}
Content-Type: multipart/form-data

file=<binary>
  • Returns 409 Conflict if a file already exists at that path. Delete it first or use a different name.
  • Returns 413 if the file exceeds the app’s size limit (default 50 MB).
  • Returns 415 if the file extension is not in the app’s allowed list.

Response:

{
  "ok": true,
  "fileId": "d4e5f6a7-...",
  "path": "/patient-records/john-doe-xray.jpg",
  "isPublic": false,
  "publicUrl": null
}

List files

GET /v1/vault/files                    # all files for the app
GET /v1/vault/files/{collection}       # files in a specific collection

Response:

{
  "ok": true,
  "files": [
    {
      "id": "d4e5f6a7-...",
      "collection": "patient-records",
      "filename": "john-doe-xray.jpg",
      "mimeType": "image/jpeg",
      "sizeBytes": 204800,
      "isPublic": false,
      "publicUrl": null,
      "createdAt": "2026-05-30T10:00:00Z"
    }
  ]
}

Download a file

GET /v1/vault/files/{collection}/{filename}

Returns the raw file bytes with the correct Content-Type header. Private files require a valid API key; public files can be fetched from the public URL instead.

Toggle public access

PATCH /v1/vault/files/{collection}/{filename}
Content-Type: application/json

{ "isPublic": true }

Once a file is public its URL is stable and permanent until you make it private again.

Delete a file

DELETE /v1/vault/files/{collection}/{filename}

Permanently removes the file from disk and its metadata from the database. This cannot be undone.

Public URLs

When isPublic is true, the file is accessible at:

http://api.{org}.local/vault/{app_slug}/{collection}/{filename}

No API key required. The Gateway forwards these requests to Vault, which checks the is_public flag before serving. If the flag is false, the response is 403 Forbidden regardless of the URL.

SDK usage

import { createClient } from "@nublestation/sdk";

const nuble = createClient({ url: "http://api.clinic.local", apiKey: "nbl_..." });

// Upload
const file = await nuble.vault.upload("patient-records", "john-doe-xray.jpg", blob);

// List a collection
const files = await nuble.vault.list("patient-records");

// Make public and get URL
await nuble.vault.setPublic("patient-records", "john-doe-xray.jpg", true);
// → http://api.clinic.local/vault/tasks/patient-records/john-doe-xray.jpg

// Download
const bytes = await nuble.vault.download("patient-records", "john-doe-xray.jpg");

// Delete
await nuble.vault.delete("patient-records", "john-doe-xray.jpg");

App settings

In the Console under App → Vault Settings, admins can configure:

SettingDefaultDescription
Allowed extensions(all)Whitelist of file extensions (e.g. pdf, jpg, png). Empty = allow all.
Max file size50 MBPer-file upload limit.

Storage layout on disk

Files are stored on the NubleStation host at:

/var/nuble/storage/
  {app_slug}/
    {collection}/
      {filename}

Paths are validated server-side — path traversal attempts are rejected before any filesystem operation.

Limits

  • No versioning. A file at a given path is a single version. To update a file, delete it and re-upload.
  • No total quota per app. Disk management is the host administrator’s responsibility.
  • 50 MB per file by default, configurable per app in Console.
  • LAN only. Files are not accessible from outside the organisation’s network.
