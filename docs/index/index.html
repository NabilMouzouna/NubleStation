Install

One command. No configuration files to edit by hand.

curl -sSL https://github.com/NabilMouzouna/NubleStation/releases/latest/download/install.sh | sudo bash

The installer prompts for your org name, admin email, and admin password — then every service comes up. You’ll see the console URL printed in your terminal within about 60 seconds.

What you get in one install

One-command install

Run the curl command, answer four questions, and every service is live on your LAN. No YAML files to write, no ports to forward.

Private by design

Patient and client data never leaves your premises. NubleStation runs entirely on your hardware — no cloud, no telemetry, no internet dependency at runtime.

Shared backend for all your apps

Auth, database, file storage, and frontend hosting — all accessible from one stable subdomain per app (tasks.clinic.local).

CLI and Console

Create apps from the Console, deploy frontends with nuble deploy. Everything a developer needs, nothing extra.

Database-enforced isolation

Postgres Row-Level Security means one application physically cannot read another app’s rows — not a trust problem, a database constraint.

LAN-native networking

CoreDNS resolves *.org.local on your network. Caddy reverse-proxies every subdomain. Zero internet dependency.

From zero to deployed app

  1. Install NubleStation on any Linux machine on your LAN.

    curl -sSL https://github.com/NabilMouzouna/NubleStation/releases/latest/download/install.sh | sudo bash
# Prompts: org name · org description · admin email · admin password
# Console live at: http://console.clinic.local

  2. Create an app and initialize the CLI.

    Open http://console.clinic.local, sign in, go to Apps → New App, enter tasks, and copy the API key (nbl_...).

    npm install -g @nublestation/cli

nuble init \
  --url http://api.clinic.local \
  --slug tasks \
  --key nbl_<keyId>.<secret>

  3. Deploy your frontend — it’s live immediately.

    npm run build          # produces dist/
nuble deploy           # zips dist/, uploads via Gateway, Orbit extracts it

# Open http://tasks.clinic.local  — live, no restart needed

Services

Gateway

The only LAN-exposed service. Validates API keys, signs internal requests with HMAC, and routes to the right service. All traffic goes through the Gateway.

Gateway docs →

Orbit — Deploy

Receives zipped frontend bundles from nuble deploy, extracts them to disk, and Caddy serves them instantly at {app}.{org}.local.

Orbit docs →

Blaze — Database

Multi-tenant REST database backed by Postgres with Row-Level Security enforced per app at the database layer.

Blaze docs →

Identity — Auth

Cross-app user accounts with shared-cookie single sign-on across all *.org.local subdomains and per-app, default-deny authorization.

Identity docs →

✦ Tip

Works offline. After installation, NubleStation has zero internet dependencies. Unplug the cable — the console, deployed apps, and the API keep running.
